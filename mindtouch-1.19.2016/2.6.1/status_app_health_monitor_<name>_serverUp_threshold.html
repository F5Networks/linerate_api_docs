/status/app/health/monitor/<name>/serverUp/threshold

Use

Use to view the configured number of health probes that must succeed to determine when a server that is down should be marked as up.

Parameters

name: Name of health monitor

Data Key

The data key contains the number of successes to use.

Data Type

uint32

Default Data Value

2

Default Allowed

False

Request Methods

GET

Related

REST API Reference - serverUp

CLI Reference - Show Running Config Commands

GET

curl -b cookie.jar -k https://10.1.2.3:8443/lrs/api/v1.0/status/app/health/monitor/hm-01/serverUp/threshold
