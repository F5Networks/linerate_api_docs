/config/app/health/monitor/<name>/invert

Use

Use to reverse (invert) how a health monitor detects whether a probe was successful. For example, if a probe would be considered successful with invert disabled, the same probe would be deemed unsuccessful with invert enabled.

Parameters

name: Name of health monitor

Data Key

The data key contains one of the values defined below in Data Values.

Data Type

uint32

Data Values

0 (disabled)

1 (enabled)

Default Data Value

0

Default Allowed

True

Request Methods

GET, PUT

Related

REST API Reference - /config/app/health/monitor

CLI Reference - Health Monitor
